# Claude Code 実装ルール

このドキュメントは、Claude Code（CC）が実装を行う際の必須ルールを定義します。

---

## 実装後の必須手順

### 1. 検証スクリプトの実行

実装完了後、**必ず**以下を実行してください：

```bash
bash verify.sh
```

このスクリプトは以下を自動的に検証します：
- ✅ 静的検証（構文、Lint、型チェック）
- ✅ 実行検証（ビルド、サーバー起動）
- ✅ 機能検証（テスト実行）
- ✅ テンプレート固有の検証（ワークフロー、ドキュメント）

---

### 2. 実行結果の全文報告

検証スクリプト実行後、以下を**全て**含めて報告してください：

#### 必須項目

##### a. 実行コマンド

```bash
bash verify.sh
```

##### b. 実行結果（全文）

`verification.log` の内容を**全文**貼り付けてください。

```
=== CC実装検証開始 ===
2025-01-13 10:30:00

【1/3】静的検証
→ npm install チェック
✅ 静的検証クリア

【2/3】実行検証
→ ビルド実行
✅ ビルド成功
→ サーバー起動テスト
✅ サーバー起動確認（ポート3000）
✅ 実行検証クリア

【3/3】機能検証
→ ユニットテスト実行
✅ ユニットテスト成功
✅ 機能検証クリア

【4/4】テンプレート固有の検証
→ GitHub Actions ワークフロー構文チェック
→ 設定ファイルサンプルの存在確認
✅ テンプレート検証クリア

=== 🎉 全検証完了 ===
```

##### c. 成否判定

- 静的検証: ✅ / ❌
- 実行検証: ✅ / ❌
- 機能検証: ✅ / ❌
- テンプレート検証: ✅ / ❌

##### d. エラーがある場合

- エラーメッセージ（全文）
- 該当ファイル・行数
- スタックトレース

##### e. スクリーンショット（UI実装の場合）

- `screenshots/result.png` として保存
- 主要画面を撮影
- ブラウザのDevToolsで主要DOM要素の存在を確認

```javascript
// Chrome DevTools Console で確認
document.querySelector('#app')  // <div id="app">...</div> ✓
document.querySelector('.header')  // <header>...</header> ✓
```

---

## 禁止事項

以下の報告は**却下**されます：

- ❌ 「動くはずです」
- ❌ 「確認しました」（証拠なし）
- ❌ 「修正しました」（実行結果なし）
- ❌ 「問題ありません」（検証なし）
- ❌ 「テストに合格しました」（テスト結果の貼付なし）

**実行結果の全文貼付がない報告は、すべて却下されます。**

## 検証失敗時の対応

### エラーが発生した場合

1. **エラーメッセージを確認**
2. **該当箇所を修正**
3. **再度 `bash verify.sh` を実行**
4. **修正内容と再実行結果を報告**

### 修正報告のフォーマット

```markdown
## 修正報告

### 発生したエラー

```
[元のエラーメッセージ]
```

### 修正内容

- ファイル: `src/xxx.js`
- 変更箇所: 45行目
- 修正内容: xxxをyyyに変更

### 再実行結果

```bash
bash verify.sh
```

```
[再実行後の verification.log 全文]
```

### 成否判定

- 静的検証: ✅
- 実行検証: ✅
- 機能検証: ✅
- テンプレート検証: ✅
```
