# API設計チェックリスト

## エンドポイント設計

- [ ] RESTful な命名規則に従っているか
- [ ] HTTPメソッドが適切か（GET/POST/PUT/PATCH/DELETE）
- [ ] URLパスはリソースを表現しているか
- [ ] バージョニング（/api/v1/）が考慮されているか

## リクエスト

- [ ] 必須パラメータのバリデーションがあるか
- [ ] 入力値の型チェックがあるか
- [ ] 入力値の範囲チェックがあるか
- [ ] ページネーションが実装されているか（大量データの場合）

## レスポンス

- [ ] 適切なHTTPステータスコードを返しているか
- [ ] エラーレスポンスの形式が統一されているか
- [ ] 成功レスポンスの形式が統一されているか
- [ ] 不要なフィールドを返していないか

## エラーハンドリング

- [ ] 予期しないエラーがキャッチされているか
- [ ] エラーメッセージがユーザーフレンドリーか
- [ ] 内部エラーの詳細が漏れていないか
- [ ] エラーログが記録されているか

## パフォーマンス

- [ ] N+1 問題が発生していないか
- [ ] 適切なキャッシュ戦略があるか
- [ ] 重い処理は非同期化されているか
- [ ] タイムアウト設定があるか

## ドキュメント

- [ ] OpenAPI/Swagger 仕様が存在するか
- [ ] 各エンドポイントの説明があるか
- [ ] リクエスト/レスポンスの例があるか
