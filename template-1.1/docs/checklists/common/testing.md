# テストチェックリスト

## 単体テスト

- [ ] ビジネスロジックのテストがあるか
- [ ] 正常系のテストがあるか
- [ ] 異常系（エラーケース）のテストがあるか
- [ ] 境界値のテストがあるか
- [ ] モック/スタブが適切に使われているか

## 統合テスト

- [ ] API エンドポイントのテストがあるか
- [ ] データベース操作のテストがあるか
- [ ] 外部サービス連携のテストがあるか

## E2Eテスト

- [ ] 主要なユーザーフローのテストがあるか
- [ ] クリティカルパスのテストがあるか

## テスト品質

- [ ] テストが独立しているか（他のテストに依存しない）
- [ ] テストデータが適切にセットアップ/クリーンアップされるか
- [ ] テストが決定的か（同じ結果が再現される）
- [ ] テスト名が何をテストしているか明確か

## カバレッジ

- [ ] クリティカルなコードパスがカバーされているか
- [ ] エッジケースがカバーされているか
- [ ] カバレッジ目標（例: 80%）を満たしているか

## CI/CD

- [ ] テストがCIで自動実行されるか
- [ ] テスト失敗時にデプロイがブロックされるか
- [ ] テスト実行時間が許容範囲内か
