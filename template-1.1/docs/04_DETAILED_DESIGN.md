### P-002: ログイン

**URL**: `/auth/login`
**認証**: 不要

#### 画面構成

```
┌─────────────────────────────────────────────────────────┐
│ Header（シンプル版：ロゴのみ）                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                  ┌─────────────────┐                    │
│                  │   ログイン      │                    │
│                  │                 │                    │
│                  │ [メールアドレス]│                    │
│                  │ [パスワード]    │                    │
│                  │                 │                    │
│                  │ [ログインボタン]│                    │
│                  │                 │                    │
│                  │ パスワードを忘れた│                   │
│                  │ アカウント作成   │                    │
│                  └─────────────────┘                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

#### UI要素

| 要素 | 種類 | バリデーション | エラーメッセージ |
|------|------|---------------|-----------------|
| メールアドレス | input[type=email] | 必須、メール形式 | 「メールアドレスを入力してください」 |
| パスワード | input[type=password] | 必須、8文字以上 | 「パスワードを入力してください」 |
| ログインボタン | button | - | - |

#### 動作フロー

```
1. ユーザーがメールアドレス・パスワードを入力
2. 「ログイン」ボタンをクリック
3. バリデーション実行
   - NG → エラーメッセージ表示（各フィールドの下）
   - OK → 4へ
4. API呼び出し（POST /api/auth/login）
   - ローディング表示（ボタンをdisabled + スピナー）
5. レスポンス処理
   - 成功 → /dashboard に遷移
   - 失敗 → エラートースト表示
```

#### エラーハンドリング

| エラー | 表示 |
|-------|------|
| 認証失敗 | 「メールアドレスまたはパスワードが正しくありません」 |
| サーバーエラー | 「エラーが発生しました。しばらく経ってから再度お試しください」 |
| ネットワークエラー | 「通信エラーが発生しました。インターネット接続を確認してください」 |

## API詳細設計

### A-001: ログイン

**エンドポイント**: `POST /api/auth/login`
**認証**: 不要

#### リクエスト

**Headers**:
```
Content-Type: application/json
```

**Body**:
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

#### バリデーション

| フィールド | ルール | エラーコード |
|-----------|-------|-------------|
| email | 必須、メール形式 | INVALID_EMAIL |
| password | 必須、8文字以上 | INVALID_PASSWORD |

#### レスポンス

**成功（200）**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "expires_in": 3600,
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "ユーザー名",
    "role": "user"
  }
}
```

**失敗（401）**:
```json
{
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "メールアドレスまたはパスワードが正しくありません"
  }
}
```

**失敗（422）**:
```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "入力内容に問題があります",
    "details": [
      { "field": "email", "message": "正しいメールアドレスを入力してください" }
    ]
  }
}
```

### A-002: 新規登録

**エンドポイント**: `POST /api/auth/register`
**認証**: 不要

#### リクエスト

**Body**:
```json
{
  "name": "ユーザー名",
  "email": "user@example.com",
  "password": "password123"
}
```

#### バリデーション

| フィールド | ルール | エラーコード |
|-----------|-------|-------------|
| name | 必須、100文字以内 | INVALID_NAME |
| email | 必須、メール形式、未登録 | INVALID_EMAIL / EMAIL_EXISTS |
| password | 必須、8文字以上 | INVALID_PASSWORD |

#### レスポンス

**成功（201）**:
```json
{
  "message": "登録が完了しました",
  "user": {
    "id": "uuid",
    "email": "user@example.com",
    "name": "ユーザー名"
  }
}
```

**失敗（409）**:
```json
{
  "error": {
    "code": "EMAIL_EXISTS",
    "message": "このメールアドレスは既に登録されています"
  }
}
```

## 共通仕様

### エラーレスポンス形式

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "ユーザー向けメッセージ",
    "details": []  // オプション：詳細情報
  }
}
```

### 認証

- JWTトークンを使用
- Authorizationヘッダーで送信: `Authorization: Bearer {token}`
- トークン有効期限: 1時間
- リフレッシュトークン: 7日間

### ページネーション

```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 100,
    "total_pages": 5
  }
}
```
